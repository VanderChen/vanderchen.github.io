Instruction for Jupyter in MSU HPCC

Posted on 2022-01-27

Install Anaconda via SSH

Same as other Linux platform, an Anaconda Installer needs to be stored in home directory. The home directory will be mounted to the host automatically.

- For the convenience of usage, installing Anaconda in Jupyter is not recommended. Since the host will change for each task applied and the system execution results in Jupyter are not persistent.

Anaconda can be install as following command line.

- Do not use sudo
- The auto init is recommended, which is [no] by default

To activate the conda environment for the first time, source .bashrc or restart the terminal is all you need.

Install miniGAP

Following the Introduction in the official Github, we install minigap from the command line.

CPU version

A conda environment named minigap is created. Additionally, Jupyter needs to be installed manually.

GPU version (TODO: still existing something wrong)

Due the lack of root privilege, we cannot install cuda from NVIDIA official web.

An alternative method is loading module which provided by MSU HPCC.

We check the provided module (Some of them listed but do not work), and load the modules we need.

Due to version issues, we make modifications on the original yml file and save as environment-gpu-custom.yml.

Then, we install from environment .yml file.

Usage of Jupyter

We use Jupyter from Open OnDemand. [ Connection to HPCC OnDemand ]

  1. choose Jupyter Notebook in Interactive Apps

  2. config the notebook and specify the anaconda path in Advance Options

  3. Submit apply.

  4. Connect to Jupyter Notebook.

Special for miniGAP

Due to the relative path in the provided notebook (sys.path.append('../code')), all the .ipynb files should be put in the notebooks directory.

Then, click the notebook you need, and enjoy your studying.
